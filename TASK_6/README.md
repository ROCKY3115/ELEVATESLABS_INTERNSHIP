
# 📊 Task 6 – Sales Trend Analysis Using SQL Aggregations

## 📌 About

This project was completed as **Task 6** of the **Elevate Labs Data Analyst Internship**. The objective of this task is to analyze sales trends using SQL aggregate functions and generate meaningful business insights from the provided sales dataset.

## 📈 Analysis Performed

- Created the sales table.
- Imported the dataset.
- Verified imported records.
- Formatted date values.
- Calculated monthly revenue using `SUM()`.
- Counted monthly orders using `COUNT(DISTINCT)`.
- Identified the top 3 revenue-generating months.
- Compared monthly sales across years.
- Calculated Monthly Average Order Value (AOV).
- Ranked months based on AOV.
- Analyzed product-line revenue trends.
- Performed seasonality analysis.

---

## ▶️ How to Run

1. Download or clone this repository.
2. Open MySQL Workbench (or PostgreSQL/SQLite).
3. Create a new database.
4. Import the `sales_data_sample.csv` dataset into a table.
5. Open the `queries.sql` file.
6. Execute each SQL query one by one.
7. Compare the generated outputs with the screenshots provided in this repository.

---

## 📊 Key Insights

- Monthly revenue varies significantly throughout the year.
- A small number of months contribute a large portion of total revenue.
- Average Order Value changes across different months.
- Product line performance differs by season.
- Ranking helps identify the best-performing months.
